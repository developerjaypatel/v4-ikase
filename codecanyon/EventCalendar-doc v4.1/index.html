Introduction

Event Calendar is a plugin that you can use on your own projects. This plugin is fully compatible with bootstrap 5. So if you plan to use it as a feature on your website or Backoffice you'll be able to implement it without any problems.

File Structure

This plugin already comes with a default bootstrap html template that you can use for your projects. But this is the file structure:

What is inside each folder?

  • css folder - All css files used on this project (plugin and website template).

  • font-awesome folder - Font used on the website template.

  • fonts folder - All fonts files used on this project (plugin and website template).

  • functions folder - Folder for the existing PHP/MYSQL functions.

  • images folder - Folder with images for DataTables and ColorPicker.

  • img folder -Images used on the website template.

  • js folder - All js files used on this project (plugin and website template).

  • license folder - All licenses to the use of the FullCalendar.

  • uploads folder - Folder which will get all image uploads from the created events.

  • lang folder - Folder with all the javascript language settings.

Installation

  1. Download the plugin from Themeforest;
  2. Unzip it into your computer;
  3. Open the folder, and select the files from it and upload them into your Webserver, just by using an FTP program like Filezilla (for MAC and Windows);
  4. These files should be uploaded into the folder you plan to use, or you can use this same extracted folder name or you can rename it.
  5. Now that you've already uploaded Event Calendar into your webserver (or Local Server), you just need to navigate to the folder you've upload, that should be something like http://yourwebsitename/eventcalendar/.
  6. Now that you've come to here, just follow these next steps.
    • If you are integrating this plugin on some other work you might be working on, just open the default MYSQL Database you are currently using and import the database .SQL file, or copy/paste the database file content and paste it into your own created database.
    • For setting the connection to your database you need to go to assets/functions/config.php and make the proper changes on the following lines:

      • Now, you just need to replace the following items:

      • Line 4 - Set your server database username.
      • Line 5 - Set your server database password.
      • Line 6 - Set your server database Hostname. Sometimes this can be the IP.
      • Line 7 - This is the database name you've created previously.

    • Well... installation's done, and now everything should be prepared for you to use the Event Calendar for the first time.
  7. Added a functionality for changing language according to your own language settings.
  8. Changing this is quite simple:
    1. Open index.php and go to line 636. There you'll be having the language type that is defined.
    2. Now you need to replace this with the default language js file you want to use. (For example if you want to use the Spanish js file, you should rename src='assets/lang/en.js' with src='assets/lang/en.js').
    3. After this you need to change also the language type to be defined on EventCalendar under assets/functions/functions.php under line 85. You need to rename that according to the language type you want to use. For example if you want to use the Spanish language version, you should rename that with es so it becomes like lang: 'en',.
    4. Easy as this :)
  9. Added a method for saving Drag and Drop Events.
    • This setting was fixed. Now it will get the database connection from the config.php file with no need of setting it also on the events_update.php file.
  10. Edit Events.
  11. Popover View.
    • Notice that you can have both views on Event Calendar (Both functions have comments for helping you changing what you want).
    • For displaying only the Popover view, you need to have the eventRender function active.
    • For displaying only the Modal Box view, you need to have the eventClick function active.
  12. NEW SETTING - Create New event with Calendar day Click.
    • This feature has been added as the Selectable date to create events under funtions.php from line 137 to line 150.
    • For having the date selector under your calendar with a click on it, you need to have the selectable: true, active.
  13. NEW SETTING - Cron Job for Past Events Deleting
    • For setting this to work you need to use the following cron job on your cPanel:

      • /usr/local/bin/php /home/johndoe/public_html/path/to/cron/script

      • For more detail you can visite the following link
    • You need to go to del_event.php and replace the database conection with your own credentials the same way you changed under config.php.
  14. NEW SETTING - Display events as ticket events
